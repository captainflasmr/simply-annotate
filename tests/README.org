* Simply Annotate Testing Process/Strategy

This file contains a description of my testing strategy for the =simply-annotate= Emacs package and the tests directory includes local unit tests and files to aid in robustly testing before pushing to =github= https://github.com/captainflasmr/simply-annotate and then on to MELPA.

* Automated test suite (ERT)

The package ships an ERT suite in =tests/simply-annotate-tests.el= driven by a =Makefile= at the project root.  The tests are batch-safe (no window system, no user interaction) and never touch the real annotation database -- file-based tests use temporary files, and the tilde-expansion test runs against a throwaway =HOME=.

** Running locally

#+begin_src bash
make test       # run the ERT suite
make compile    # byte-compile, treating warnings as errors
make checkdoc   # run checkdoc on simply-annotate.el
make lint       # run package-lint (after `make deps')
make deps       # install transient + package-lint into ./.elpa
make all        # compile + test + checkdoc
#+end_src

Target a specific Emacs binary with, e.g. =make EMACS=emacs-29.4 test=.

Each target loads =tests/init.el= first, which points =package-user-dir= at a project-local =.elpa= (so dependency installs never disturb your real package store) and puts the working copy at the front of =load-path= (so it shadows any MELPA-installed =simply-annotate=).  Since the package requires Emacs 28.1+, the bundled =transient= is always available and =make deps= is a no-op for the test matrix.

** Coverage

The suite covers the logic that is meaningfully unit-testable in batch:

- *Helpers* -- timestamp format, thread vs string detection, annotation text/summary extraction, boilerplate stripping, Info-key detection.
- *Defaults* -- configured default status/priority resolution, and that the stock lists still lead with =open= / =normal= (issue #9).
- *Threads* -- thread creation, replies (root and nested), legacy comment-id migration, comment-tree reconstruction, status/priority validation, tag add/remove, tag collection and filtering.
- *Database* -- save/read round-trip, add/remove update, primary-wins merge, and tilde-path expansion (issue #10).
- *Overlays* -- property/span recording, full clear, and auxiliary-overlay cleanup on removal in text styles (issue #7 regression).
- *Serialization* -- serialize/deserialize round-trip, stale detection on edit, and relocation on moved text.
- *Kanban / counts* -- custom-status visibility and legacy/plain-string fallback into the first column, status tallies (issue #9 regression).
- *Keymaps* -- command-map and repeat-map wiring (issue #8).

* Continuous integration

=.github/workflows/ci.yml= runs the Makefile on every push and pull request:

- A =test= matrix across Emacs 28.2, 29.4, 30.1, and =snapshot= (snapshot is allowed to fail), each running =make deps compile test checkdoc=.
- A separate =lint= job running =package-lint= on a stable Emacs.

* Validate/lint elisp

- Run =checkdoc=
- =byte-compile= on simply-annotate.el
- =flycheck= on simply-annotate.el
- =package-lint= on simply-annotate.el
- Possibly =melpazoid= (if I can be bothered!)

** Batch-byte-compile

#+begin_src bash
#!/bin/bash

# List of elisp files to compile
elisp_files=(
  "simply-annotate.el"
)

# Directory above current one
elisp_dir="$( cd "$( dirname "${BASH_SOURCE[0]}" )/.." && pwd )"

for el in "${elisp_files[@]}"; do
  emacs -Q --batch \
    --eval "(add-to-list 'load-path \"$elisp_dir\")" \
    -f batch-byte-compile "$elisp_dir/$el"
done
#+end_src

** Run simply-annotate with different Emacs versions

- [X] emacs-28.1
- [X] emacs-29.3
- [X] emacs-29.4
- [X] emacs-30.1

Using the following script to generate a very simple init.el file which is then run up against varying versions of Emacs to then run up =simply-annotate= using =C-c o= and perform some quick tests.

#+begin_src bash
#!/bin/bash

# Script to test simply-annotate package in a vanilla Emacs environment
# Usage: ./test-simply-annotate.sh [emacs-version]
# Example: ./test-simply-annotate.sh 29.3

# Default Emacs version if none specified
VERSION=${1:-"28.2"}
PACKAGE_PATH="$HOME/source/repos/simply-annotate"

# Create a temporary directory for the test
TEMP_DIR=$(mktemp -d -t simply-annotate-test-XXXXXX)
INIT_FILE="$TEMP_DIR/init.el"

# Create a minimal init.el file
cat > "$INIT_FILE" << EOF
;; Minimal init.el for testing simply-annotate (offline version)

;; Add the package to the load path
(add-to-list 'load-path "$PACKAGE_PATH")

;; Load and configure the package
(require 'simply-annotate)
(global-set-key (kbd "C-c a") simply-annotate-command-map)
(add-hook 'find-file-hook #'simply-annotate-mode)
(add-hook 'dired-mode-hook #'simply-annotate-dired-mode)

;; Show a message to confirm we're in test mode
(message "Testing simply-annotate in vanilla Emacs %s" emacs-version)
EOF

# Run Emacs with the temporary init file
echo "Starting Emacs $VERSION with minimal configuration..."
echo "Testing simply-annotate from: $PACKAGE_PATH"
echo "Temporary init file: $INIT_FILE"

# Remove any byte-compiled file to avoid cross-version incompatibility
rm -f "$PACKAGE_PATH/simply-annotate.elc"

# Run the appropriate Emacs version
"$HOME/bin/emacs-$VERSION" -Q -l "$INIT_FILE"

# Cleanup
echo "Cleaning up temporary directory..."
rm -rf "$TEMP_DIR"
#+end_src
