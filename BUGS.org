This document contains the current bugs/improvements, this is just for me mainly but does include any Issues reported.

* Bugs/Improvements

** TODO add automated unit tests

Currently validation is manual (byte-compile, checkdoc, package-lint, flycheck). Even a basic ERT suite covering database load/save, thread creation, and overlay placement would catch regressions.

** TODO annotation buffer could support org-mode syntax

Related to Issue #3. The annotation editing buffer is plain text; optionally deriving from org-mode would give users bold, italics, links, footnotes etc.

** DONE allow user-supplied buffer key function

Related to Issue #2. Fixed in 0.9.1. Added =simply-annotate-file-key-functions= (alist mapping major modes to key generators) and =simply-annotate-file-key-function= (buffer-local override). Info-mode is supported out of the box with per-node annotation tracking.

** DONE review keymap activation approach

Related to Issues #4 and #5. Fixed in 0.9.3. Added =simply-annotate-command-map=, a standalone prefix keymap with short single-key bindings (=j=, =r=, =n=, etc.) that users bind to their own prefix key. Separate from =simply-annotate-mode-map= which uses =M-s= prefixed keys.

* Issues

Issues on the [[https://github.com/captainflasmr/simply-annotate/issues][GitHub Issues page]]

** DONE If an annotation is created outside of simply-annotate-mode, turning on the mode erases it #6

Fixed in 0.9.2. Mode activation now persists any existing annotation overlays to the database before clearing and reloading, so annotations created while the mode was off are never lost.

** DONE Minor mode activates keymap explicitly, leads to weird outcomes #5

Fixed in 0.9.3. Short-key bindings moved to =simply-annotate-command-map=, a separate prefix keymap. Enabling the minor mode no longer intercepts single-key presses during normal typing.

** DONE Right way to set key bindings to not interfere with existing bindings, like for Consult #4

Fixed in 0.9.3. =simply-annotate-command-map= can be bound to any prefix (e.g. =C-c a=) giving =C-c a j=, =C-c a r= etc. without interfering with Consult or other packages.

** OPEN Support for org syntax in annotations #3

Partially addressed: listing and export buffers use org-mode. The annotation *editing* buffer still uses fundamental-mode. Org syntax support for comment editing remains on the roadmap.

** DONE Support for non-file buffers #2

Fixed in 0.9.1. Added =simply-annotate-file-key-functions= alist and =simply-annotate-file-key-function= buffer-local variable. Info-mode supported out of the box with per-node tracking.

** DONE More complex threading system #1

Fixed in 0.9.0. Full nested/hierarchical replies with comment =id= and =parent-id= fields, hierarchical =completing-read= for reply target and comment editing, tree-line display, and automatic legacy database migration.
