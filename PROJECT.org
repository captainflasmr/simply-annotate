* Completed

** Phase 1: Per-Project Databases (v1.0.0)

- =simply-annotate-database-strategy= defcustom (=global= / =project= / =both=)
- =simply-annotate-project-file= defcustom (default =.simply-annotations.el=)
- =simply-annotate--database-path= resolution via =project.el=
- =simply-annotate--read-db= / =--merge-databases= helpers
- Strategy-aware =--load-database= and =--save-database=
- Relative file keys in project mode (=file-relative-name=)
- =simply-annotate-migrate-to-project= interactive migration command

** Phase 2: Project-Scoped Views (v1.0.0)

- =simply-annotate-show-project= (=P=) -- org listing filtered to current project
- =simply-annotate-list-project-table= (=C-t=) -- sortable table with File column
- =simply-annotate-jump-to-file= -- project-scoped by default, =C-u= for all
- Extracted =--show-filtered= and =--project-annotations= shared helpers

** Phase 3: Stale Detection & Position Tracking (v1.0.0)

- Content hashing via =sxhash-equal= at serialize time (=text-hash= field)
- Stale detection at deserialize: compares stored hash vs current buffer text
- =[STALE]= markers in inline display, tooltips, and table views (Stale column)
- =simply-annotate-stale-face= (tomato, italic)
- Text context storage (=text-context= field, up to 150 chars)
- Automatic position relocation: searches buffer for stored context on hash mismatch
- =[MOVED]= marker for relocated annotations, clears on next save
- Stale hash preserved across saves/reverts (not silently overwritten)

** Other (v1.0.0)

- Dired keymap fix: =simply-annotate-dired-mode= mirrors global prefix via =where-is-internal=

* Future Ideas

** Kanban Board View

Use annotation statuses (open, in-progress, resolved, closed) as kanban columns.  Display a board view showing annotations grouped by status, with the ability to drag/move between columns.  Could leverage =tabulated-list-mode= or a custom buffer rendering.

Considerations:
- Scope: per-buffer, per-project, or global
- Column customisation (user-defined statuses as columns)
- Navigation: jump to source from any card
- Status change: move annotation between columns directly from the board

** Merge Conflict Handling

Custom merge driver or Emacs-side helper for =.simply-annotations.el= conflicts during git merges.  The s-expression format makes structural merging feasible.

** Annotation Expiry / Cleanup

Auto-flag or remove annotations older than N days, or annotations on deleted code.  Would require adding a =created-at= timestamp to annotations.
