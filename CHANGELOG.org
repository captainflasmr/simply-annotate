* Versions

** <2026-03-26 Wed> *0.9.0*

Nested/hierarchical comment threading.

*** Features

- *Nested replies* -- Replies can now target any comment in a thread, not just
  the root.  When replying to a thread with multiple comments, =M-s r= prompts
  with a hierarchical =completing-read= menu showing the comment tree so you can
  pick which comment to reply to.  Single-comment threads reply to the root
  directly with no extra prompt.
- *Per-comment editing* -- Editing a thread with multiple comments (=C-u M-s j=)
  now prompts which comment to edit via a hierarchical =completing-read= menu.
  Single-comment threads edit the comment directly with no extra prompt.
- *Tree-line display* -- Nested comments are connected with vertical bar
  characters (=│=, =├=, =└=) like indent-bars or the =tree= command, making the
  reply hierarchy visually clear.
- *Hierarchical display* -- Annotation buffer, inline overlays, listing buffer,
  and org export all render nested comments with indentation and =↳= markers
  showing the reply structure.  Separator lines between comments have been
  removed as the nesting provides sufficient visual structure.
- *Database migration command* -- =M-x simply-annotate-migrate-threading= scans
  the entire database and assigns comment IDs, inferring parent-child
  relationships from the existing =type= field (replies become children of the
  root comment).  Safe to run multiple times.

*** Data model

- Each comment now carries an =id= (unique string) and =parent-id= (nil for
  root comments, or a parent comment's id for replies).
- Comments are still stored as a flat list; the tree structure is built on the
  fly for display.
- Legacy comments without =id=/=parent-id= are automatically migrated on first
  access, with =type = "reply"= comments becoming children of the root comment.

** <2026-03-26 Wed> *0.8.2*

Display styles, comment formatting, and usability refinements.

*** Features

- *Fringe bracket display style* -- New =fringe-bracket= style draws a vertical
  bracket in the fringe spanning the full annotated region.  Uses the configured
  fringe indicator (triangle) at the top with a continuous vertical bar down to a
  horizontal cap at the bottom.  Customisable via =simply-annotate-fringe-bracket-face=.
- *Tint display style* -- New =tint= style applies a subtle background colour
  derived from the current theme (lightened by =simply-annotate-tint-amount=,
  default 5%).  Adapts automatically to any theme.

*** Improvements

- *Consistent comment formatting* -- Inline display and =*Annotation*= buffer now
  use the same compact layout: =✎ [STATUS/PRIORITY]= header, =├= joined
  separators between comments, matching reply indentation, and =└───┘= footer.
- *File-level hidden at all* -- File-level annotations are suppressed when
  viewing at the =all= pseudo-level, preventing whole-buffer overlays from
  obscuring smaller annotations.

** <2026-03-25 Tue> *0.8.1*

Dired integration, display refinements, and usability improvements.

*** Features

- *Dired integration* -- =simply-annotate-dired-mode= shows fringe indicators in
  dired buffers next to files that have annotations.  Enable globally with
  =(add-hook 'dired-mode-hook #'simply-annotate-dired-mode)=.  Indicators refresh
  automatically when the dired buffer is re-read.
- *All-levels display* -- New =all= pseudo-level shows annotations from every level
  simultaneously.  Accessible via =M-s [= / =M-s ]= cycling and =M-x
  simply-annotate-set-level=.  Now the default when the mode starts.
- *Inline default* -- =simply-annotate-inline-default= customisation variable to
  enable inline annotations on mode startup.
- *Inline word-wrap* -- Inline comment text is now word-wrapped to
  =simply-annotate-inline-fill-column= (default 72) so annotations always display
  as multiline regardless of =truncate-lines= or =visual-line-mode= settings.
- *C-g to dismiss* -- =C-g= in the annotation buffer now cancels/hides, same as
  =C-c C-k=.

*** Improvements

- *Compact comment rendering* -- Removed emoji icons from comment/reply display.
  Comments show author name directly; replies are indented.
- *No empty meta line* -- Annotations without an author (e.g. auto-generated) no
  longer produce a blank line before the comment body in inline display.
- *Header position embedded* -- Current annotation position is now shown within the
  active level count (e.g. =DEFUN: 3/5=) rather than as a separate indicator.

** <2026-03-24 Mon> *0.8.0*

Org-mode annotation listing with folding, global overview, and live refresh.

*** Features

- *Org-mode listing* -- =M-s l= now produces an org-mode buffer with full folding
  support.  Annotation levels are level-1 headings, each annotation is a level-2
  heading, and the annotated region and comments/replies each get their own
  sub-headings.  Use =TAB= / =S-TAB= for folding.
- *Global annotation overview* -- =simply-annotate-show-all= displays all annotations
  across all files in a single org-mode buffer.  Files are top-level headings that
  expand into level groups and individual annotations.  Opens with =content= visibility
  showing all headings.
- *Quick file jump* -- =simply-annotate-jump-to-file= provides the previous
  completing-read navigation for quick file switching.  Bind globally for fast access.
- *Listing navigation* -- =n= / =p= step between annotation headings in the listing
  buffer and automatically jump to the corresponding source location.  =RET= jumps to
  source from anywhere within an annotation.  =q= quits.
- *Live listing refresh* -- The listing buffer automatically updates when annotations
  are saved, edited, replied to, removed, or have status/priority/tags/author changed.

*** Improvements

- *Removed =M-s 0= from mode map* -- =simply-annotate-show-all= operates on the
  database rather than the current buffer, so it belongs as a global binding.
  Recommended: =(global-set-key (kbd "C-c 0") #'simply-annotate-show-all)=.

*** Key binding changes

- =M-s 0=: Removed from mode map (use global binding for =simply-annotate-show-all=)
- Listing buffer: =n= / =p= / =RET= / =q= for navigation

** <2026-03-23 Sun> *0.7.1*

Inline display, subtle style, and annotation listing improvements.

*** Features

- *Inline display toggle* -- =M-s /= toggles box-drawn annotation text blocks on
  top of any display style.  Shows full multiline annotation content with thread
  metadata, author info, and comment history directly in the buffer.
- *Inline positioning* -- =simply-annotate-inline-position= controls where blocks
  appear: ='after= (below annotated text) or ='above= (above annotated text).
- *Subtle display style* -- New display style using overline/underline to bracket
  annotated regions.  Visible but unobtrusive, ideal for pairing with inline display.
- *Level-filtered display* -- All display styles now show only annotations matching
  the active level.  Non-matching annotations have styling removed but text remains.
- *Header level counts* -- Header-line shows annotation counts for every level
  (e.g. =FILE:2 | DEFUN:5 | LINE:3=) with the active level in bold.
- *Bidirectional level cycling* -- =M-s ]= cycles levels forward, =M-s [= cycles
  backward.

*** Improvements

- *Annotation listing* -- Redesigned =M-s l= listing with per-level sections, counts
  per level in the header, cleaner formatting, and source context for each annotation.
- *Listing auto-level sync* -- Jumping from the annotation listing to a source
  location automatically switches the source buffer to the annotation's level.
- *View mode focus* -- =M-s j= on annotated text now shows the annotation buffer
  without stealing focus from the source buffer.  Edit, reply, and sexp modes
  still select the annotation buffer.
- *Reply fix* -- Fixed reply mode variable being wiped by =kill-all-local-variables=
  during mode switch, which caused replies to overwrite the original comment.
- *Inline faces* -- =simply-annotate-inline-face= (body text) and
  =simply-annotate-inline-border-face= (box-drawing characters) for customisation.

*** Key binding changes

- =M-s /=: Toggle inline annotation display (new)
- =M-s '=: Cycle display style (moved from =M-s ]=)
- =M-s ]=: Cycle annotation level forward (moved from display style)
- =M-s [=: Cycle annotation level backward (was forward-only)

** <2026-03-23 Sun> *0.7.0*

Annotation levels -- layered annotations for progressive code exploration.

*** Features

- *Annotation levels* -- Three annotation levels: =file= (whole-file description),
  =defun= (function/element description), and =line= (individual syntactical
  elements).  Each annotation is tagged with a level when created, inheriting the
  currently active view level.  All overlay markers remain visible regardless of
  level; only interaction (navigation, editing, header display) is filtered to the
  active level.
- *Level cycling* -- Press =M-s [= to cycle through levels: =file= -> =defun= ->
  =line=.  Symmetric with =M-s ]= for display style cycling.
- *Direct level selection* -- =M-x simply-annotate-set-level= to jump to a
  specific level via completing-read.
- *Header level indicator* -- Header-line now shows the active level as =[FILE]=,
  =[DEFUN]=, or =[LINE]= with annotation count scoped to that level.
- *Backward-compatible storage* -- Existing annotations without a level default to
  =defun=.  No database migration required.

*** Improvements

- *Internal naming convention* -- All private functions now use the =simply-annotate--=
  double-dash prefix, clarifying the public API boundary.
- *Buffer-local state* -- Source-buffer variables (draft overlay, header state) are
  now buffer-local, preventing cross-buffer contamination in multi-buffer workflows.
- *Thread ID robustness* -- Thread IDs now include a timestamp component, reducing
  collision risk.
- *Database error reporting* -- Corrupted database files now produce a warning
  message instead of being silently ignored.
- *Single database load per update* -- =simply-annotate--update-database= no longer
  reads the database file twice.
- *defcustom correctness* -- Fixed =:type= mismatches for highlight face and
  separator customization variables.  Added missing =:group= to fringe indicator.
- Removed =C-g= override in annotation buffer that hijacked =keyboard-quit=.
- Added =;;; simply-annotate.el ends here= footer.
- Version header updated to match actual release.
